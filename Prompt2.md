# Gemini 繪圖 Prompt - 工作流程助理架構圖（方案 C：混合方案）

> **用途**：使用 Gemini 生成「工作流程助理」的混合架構流程圖（Lists → 文件轉換 → Generative Answers）  
> **日期**：2026-03-04

---

## 🎨 Gemini Prompt

```
請繪製一個專業的系統架構流程圖，展示「工作流程助理」AI Agent 的三階段混合架構。

【圖表要求】
- 風格：現代化、科技感、清晰易懂
- 配色：使用藍色系（主要）、綠色（成功）、橙色（強調）、紫色（轉換）
- 排版：垂直流程佈局（由上而下），三個階段清晰分層
- 圖示：使用現代化的 icon（對話、資料庫、文件、自動化等）

【流程架構：三階段循環】

═══════════════════════════════════════════════════════════════

【階段 1：資料輸入流程】（頂部）

標題：「📝 階段 1：資料收集與儲存」

流程步驟：
1. 「使用者」（人物 icon）
   ↓ 自然語言對話
2. 「Copilot Studio Agent」（AI 對話 icon）
   - 名稱：工作流程助理
   - 功能：引導式對話收集
   ↓ 呼叫 Power Automate
3. 「Power Automate Flow」（齒輪 icon）
   - Flow 1：建立工作主檔
   - Flow 2：建立流程步驟
   ↓ 寫入結構化資料
4. 「SharePoint Lists」（列表/資料庫 icon）
   - List A：工作項目主檔
   - List B：工作流程步驟
   
標註：「✅ 結構化資料儲存」

═══════════════════════════════════════════════════════════════

【階段 2：資料轉換流程】（中間）

標題：「🔄 階段 2：自動格式轉換（定期執行）」

流程步驟：
1. 「排程觸發器」（時鐘 icon）
   - 每日 or 每小時執行
   ↓ 觸發 Flow
2. 「Power Automate Flow」（轉換/處理 icon）
   - Flow 3：Lists 資料轉換
   ↓ 讀取 Lists
3. 「SharePoint Lists」（列表 icon，與階段 1 相同）
   - 讀取所有流程資料
   ↓ 轉換處理
4. 「資料處理引擎」（齒輪/處理 icon）
   - 組合主檔 + 步驟
   - 產生格式化文本
   - 格式：Word / Markdown
   ↓ 儲存文件
5. 「SharePoint 文件庫」（文件/資料夾 icon）
   - 工作流程知識庫.docx
   - 工作流程知識庫.md
   
標註：
- 步驟 2→3：「自動讀取」
- 步驟 4：「Lists → 文件轉換」
- 步驟 5：「可被 AI 索引的格式」

特別標示：在此階段旁邊加入說明框
「💡 為什麼要轉換？
Copilot Studio 的 Generative Answers 無法直接讀取 Lists，
但可以讀取 SharePoint 文件（Word, Markdown）。」

═══════════════════════════════════════════════════════════════

【階段 3：知識庫問答流程】（底部）

標題：「💬 階段 3：AI 知識庫問答」

流程步驟：
1. 「使用者」（人物 icon）
   - 提問：「XXX 流程怎麼做？」
   ↓ 自然語言問題
2. 「Copilot Studio Agent」（AI 對話 icon）
   - Generative Answers 啟動
   ↓ 搜尋知識來源
3. 「SharePoint 文件庫」（文件 icon，與階段 2 輸出相同）
   - AI 自動搜尋文件內容
   - 找到相關流程段落
   ↓ 回傳相關內容
4. 「AI 語言模型」（大腦/AI icon）
   - 理解文件內容
   - 生成自然語言摘要
   ↓ 生成回答
5. 「Copilot Studio Agent」（AI 對話 icon）
   - 以友善語氣呈現
   ↓ 自然語言回覆
6. 「使用者」（人物 icon）
   - 收到清楚的流程說明
   
標註：
- 步驟 2→3：「語義搜尋（Semantic Search）」
- 步驟 3→4：「文件內容提取」
- 步驟 4→5：「AI 自動摘要」

═══════════════════════════════════════════════════════════════

【整體架構設計】

1. 三階段用不同顏色的區塊框起來：
   - 階段 1：藍色框
   - 階段 2：紫色框（強調轉換）
   - 階段 3：綠色框

2. 資料流向箭頭：
   - 階段 1 → 階段 2：粗虛線箭頭（定期同步）
   - 階段 2 → 階段 3：實線箭頭（知識來源）
   - 在箭頭旁標註：
     「階段 1→2：定期同步（每日/每小時）」
     「階段 2→3：即時搜尋」

3. 關鍵優勢標註（在圖表右側加入說明框）：
   「✨ 混合架構優勢：
   ✅ 結構化儲存（Lists）：易於管理、查詢、統計
   ✅ 文件格式（Word/MD）：可被 AI 理解、語義搜尋
   ✅ 自動同步：資料更新後自動轉換
   ✅ 自然語言問答：使用者友善、智能摘要」

4. 圖表底部加入完整資料流說明：
   「📊 完整資料流：
   使用者對話 → 結構化儲存(Lists) → 自動轉換(文件) → AI 語義搜尋 → 智能回答」

【技術標籤】（顯示於圖表角落）
- Microsoft Copilot Studio (Generative Answers)
- Power Automate (3 Flows)
- SharePoint Lists (結構化儲存)
- SharePoint 文件庫 (AI 知識來源)
- 自然語言處理 (NLP)
- 語義搜尋 (Semantic Search)

【時間軸標註】（可選，在圖表左側）
- T0：使用者建立流程
- T1：資料寫入 Lists（即時）
- T2：排程轉換執行（定期，如每小時）
- T3：文件更新完成
- T4：使用者可查詢最新資料

請繪製清晰、專業且易於理解的三階段架構圖，強調資料從結構化到文件的轉換過程。
```

## 📋 補充說明

### 階段 2 資料轉換細節

**Flow 3 處理邏輯：**
```yaml
觸發器: Recurrence (排程)
頻率: 每日 02:00 或每小時

步驟 1: Get items from WorkCatalog
步驟 2: Apply to each (遍歷每個流程)
  → Get items from WorkStep (Lookup 該流程的步驟)
  → 組合成 Markdown 格式：
    # 【流程名稱】
    - 部門：XXX
    - 負責：XXX
    
    ## 步驟
    1. 步驟名稱
       - 操作：XXX
       - 工具：XXX
    ...
步驟 3: Combine all text (合併所有流程)
步驟 4: Create file in SharePoint
  → 檔名：工作流程知識庫_YYYYMMDD.md
  → 或更新現有檔案
```

**文件格式範例（Markdown）：**
```markdown
# 工作流程知識庫

更新時間：2026-03-04 14:00

---

## 【產品出貨流程】

**部門**：製造部  
**負責角色**：倉管人員  
**工作目的**：確保產品從工廠順利出貨給客戶

### 流程步驟

#### 步驟 1：登入 SAP 系統
- **操作說明**：使用出貨專用帳號登入
- **使用工具**：SAP
- **注意事項**：確認帳號權限

#### 步驟 2：建立出貨單
- **操作說明**：填寫客戶編號、產品型號、數量
- **使用工具**：SAP
- **產出結果**：出貨單號

---

## 【新人報到流程】
...
```
