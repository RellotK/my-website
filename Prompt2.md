# Gemini 繪圖 Prompt - 工作流程助理架構圖（方案 C：混合方案）

> **用途**：使用 Gemini 生成「工作流程助理」的混合架構流程圖（Lists → 文件轉換 → Generative Answers）  
> **日期**：2026-03-04

---

## 🎨 Gemini Prompt（精簡版）

```
請繪製一個簡潔專業的系統架構流程圖，展示「工作流程助理」的三階段資料流。

【圖表要求】
- 風格：簡潔、現代、清晰
- 配色：藍色（主要）、紫色（轉換）、綠色（問答）
- 排版：垂直三層，由上而下
- 圖示：簡單的 icon，不要過多文字

【三階段架構】

┌────────────────────────────────────┐
│ 📝 階段 1：資料填寫                 │
│ （藍色區塊）                        │
│                                    │
│ 使用者 → Agent → Flow → SharePoint │
│                        Lists       │
└───────────────┬────────────────────┘
                │
                │ 定期同步
                ↓
┌────────────────────────────────────┐
│ 🔄 階段 2：資料轉換                 │
│ （紫色區塊）                        │
│                                    │
│ 排程 → Flow → Lists → 文件庫       │
│              (讀取)  (Word/MD)     │
└───────────────┬────────────────────┘
                │
                │ 知識來源
                ↓
┌────────────────────────────────────┐
│ 💬 階段 3：知識問答                 │
│ （綠色區塊）                        │
│                                    │
│ 使用者 → Agent → 文件庫 → AI回覆   │
│        (Generative Answers)        │
└────────────────────────────────────┘

【繪圖要點】

階段 1 - 資料填寫：
- 使用者（人物 icon）
- Agent（對話泡泡 icon）
- Flow（齒輪 icon）
- SharePoint Lists（列表 icon）
- 箭頭標註：「對話」→「自動化」→「儲存」

階段 2 - 資料轉換：
- 排程觸發（時鐘 icon）
- Flow（轉換 icon）
- Lists（與階段 1 相同）
- 文件庫（文件 icon）
- 箭頭標註：「定期」→「讀取」→「轉換」

階段 3 - 知識問答：
- 使用者（人物 icon）
- Agent（AI icon）
- 文件庫（與階段 2 輸出相同）
- 箭頭標註：「提問」→「搜尋」→「回答」

【整體設計】
1. 三階段用不同顏色區塊區分
2. 階段之間用虛線箭頭連接
3. SharePoint Lists 和文件庫的位置要清楚分開
4. 不要加太多說明文字，保持簡潔

請繪製一個一目了然的三階段流程圖。
```

## 📋 補充說明

### 階段 2 資料轉換細節

**Flow 3 處理邏輯：**
```yaml
觸發器: Recurrence (排程)
頻率: 每日 02:00 或每小時

步驟 1: Get items from WorkCatalog
步驟 2: Apply to each (遍歷每個流程)
  → Get items from WorkStep (Lookup 該流程的步驟)
  → 組合成 Markdown 格式：
    # 【流程名稱】
    - 部門：XXX
    - 負責：XXX
    
    ## 步驟
    1. 步驟名稱
       - 操作：XXX
       - 工具：XXX
    ...
步驟 3: Combine all text (合併所有流程)
步驟 4: Create file in SharePoint
  → 檔名：工作流程知識庫_YYYYMMDD.md
  → 或更新現有檔案
```

**文件格式範例（Markdown）：**
```markdown
# 工作流程知識庫

更新時間：2026-03-04 14:00

---

## 【產品出貨流程】

**部門**：製造部  
**負責角色**：倉管人員  
**工作目的**：確保產品從工廠順利出貨給客戶

### 流程步驟

#### 步驟 1：登入 SAP 系統
- **操作說明**：使用出貨專用帳號登入
- **使用工具**：SAP
- **注意事項**：確認帳號權限

#### 步驟 2：建立出貨單
- **操作說明**：填寫客戶編號、產品型號、數量
- **使用工具**：SAP
- **產出結果**：出貨單號

---

## 【新人報到流程】
...
```

### 階段 3 Generative Answers 設定

**Copilot Studio 設定：**
```yaml
Knowledge Source:
  Type: SharePoint
  Site: https://unimicron33341.sharepoint.com/sites/ym-training
  Include: Document Library (文件庫)
  
Indexing:
  - Word 文件會被自動索引
  - Markdown 文件會被自動索引
  - 索引頻率：15-60 分鐘

Generative Answers 行為:
  - 當使用者問題不匹配 Topic 時觸發
  - AI 自動搜尋文件內容
  - 生成自然語言摘要
```

## 🔄 與前一個架構圖的差異

| 項目 | 前一個架構（方案 B） | 這個架構（方案 C） |

| 回答生成 | 格式化文字拼接 | AI 自動摘要 |
