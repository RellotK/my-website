# 移動端優化說明

## ✅ 已完成的優化

### 📱 響應式布局
- **三段式響應**：桌面端 → 平板端 → 手機端
- **自適應網格**：地牢網格、技能樹、背包等自動適配螢幕大小
- **流式布局**：在小螢幕上自動變為單列布局

### 🎮 虛擬方向鍵
- **只在移動端顯示**：桌面端不顯示，移動端自動顯示
- **觸摸反饋**：按下時有視覺反饋
- **位置**：在地牢網格下方，方便單手操作

### 🖱️ 觸摸優化
- **移除高亮**：點擊時不會出現藍色選取框
- **防止縮放**：禁用雙擊縮放和捏合縮放
- **橡皮筋效果**：防止頁面上下拉動
- **按鈕大小**：增大按鈕尺寸，更易點擊

### 📐 尺寸調整

#### 桌面端（>1024px）
- 三列布局：角色面板 | 地牢視圖 | 資訊面板
- 地牢格子：25px × 25px

#### 平板端（768px-1024px）
- 單列布局：所有面板垂直排列
- 地牢格子：保持 25px × 25px

#### 手機端（480px-768px）
- 單列緊湊布局
- 地牢格子：22px × 22px
- 顯示虛擬方向鍵
- 按鈕 2×2 或 1×2 排列

#### 小手機端（<480px）
- 極致緊湊布局
- 地牢格子：20px × 20px
- 所有按鈕單列排列

### 🎨 視覺優化
- **字體大小**：移動端自動縮小字體
- **間距調整**：減少不必要的空白
- **滾動優化**：模態框支援垂直滾動
- **觸摸滾動**：流暢的慣性滾動

## 📱 手機測試方法

### 方法一：瀏覽器開發者工具
1. 按 F12 打開開發者工具
2. 點擊「Toggle device toolbar」（Ctrl+Shift+M）
3. 選擇手機型號（如 iPhone SE、Samsung Galaxy S20）
4. 測試不同螢幕尺寸

### 方法二：實機測試
1. 將遊戲文件上傳到伺服器
2. 用手機瀏覽器打開網址
3. 或使用 Python 啟動本地伺服器：
   ```bash
   python -m http.server 8000
   ```
4. 手機連接同一 WiFi，訪問 `http://電腦IP:8000`

## 🎮 移動端操作說明

### 移動方式
1. **虛擬方向鍵**：點擊 ⬆️⬇️⬅️➡️ 按鈕
2. **點擊格子**：直接點擊相鄰格子移動
3. **拖曳**：不支援（避免誤操作）

### 戰鬥操作
- 所有戰鬥按鈕放大，易於點擊
- 技能按鈕自動排列
- 戰鬥畫面可滾動查看完整內容

### 技能樹
- 技能卡片單列顯示
- 易於瀏覽和選擇
- 支援垂直滾動

### 背包管理
- 物品圖示自動調整大小
- 點擊物品使用/裝備/賣出
- 滾動查看更多物品

## 🔧 已知限制

### 不支援的功能
- ❌ 鍵盤控制（移動端沒有鍵盤）
  - ✅ 解決方案：使用虛擬方向鍵或點擊移動
- ❌ 懸停效果（觸控螢幕沒有懸停）
  - ✅ 解決方案：使用點擊和長按

### 效能考慮
- 複雜動畫在低端機型可能卡頓
- 地牢網格較大時滾動可能不流暢
- 建議關閉背景應用程式以提升效能

## 💡 使用技巧

### 橫向/直向模式
- **直向模式**：推薦，UI 設計主要針對直向
- **橫向模式**：可用，但某些元素可能較小

### 省電模式
- 遊戲不會持續動畫，相對省電
- 戰鬥時會有較多互動
- 建議開啟省電模式玩遊戲

### 多指操作
- 不支援多指手勢
- 請使用單指點擊操作
- 避免雙指縮放（已禁用）
